关于Phar的总结

  • php

Phar允许使用单个文件打包应用程序, 这个文件包含运行程序所需的所有东西, 便于部署等(不同于C语言之类的编译程序, 该文件实际是一个归档文件)

创建 Phar

  • Phar需要 PHP >= 5.2
  • Phar默认为只读模式, 创建自己的phar需要设置 php.ini phar.readonly 必须设置为 0 。
  1. 创建如下目录结构
[test]
    |- src
        |- 1.php   # <?php  echo 'This is 1.php' ?>
        |- 2.php   # <?php  echo 'this is 2.php'  ?>
     |- go.php

go.php 里将写入我们的制作Phar的代码, 打包的对象是src里的文件,我们创建如下代码

$t1 = new  Phar('li9hu.phar')  // 创建一个名为lihu.phar的包
$t1 -> buildFromDirectory(dirname(__FILE__).'/src');  //将要打包的文件夹
$t1 -> setDefaultStub('1.php','2.php');

运行go.php 后将会再当前目录生产一个 li9hu.phar的文件

执行 php li9hu.phar 将得到 this is 1.php

如果php 未设置解析phar 可以创建一个php文件 里边写入 require(li9hu.phar)从而使用php的文件来执行

Stub文件

运行Phar文件时,stub文件(入口存根文件)被当做一个meta文件(描述数据的数据)来初始化Phar, 并告诉Phar文件在被调用时该做什么。这个存根文件默认是index.php, 但是可以创建两个不同的存根文件,即phar和webphar的区别。

直接在命令行执行:  php  li9hu.phar      result:  this is 1.php
使用php文件执行:   <?php include "li9hu.phar" ?>        result: this is 1.php
通过网页直接访问phar文件 http://localhost//li9hu.phar   result: thisis 2.php
(通过浏览器访问phar需要配置apache 否则将显示的是phar包里的内容)

Phar的反序列化

创建一个准备反序列化执行的Phar包

直接上demo来演示

文件a.php的内容如下

<?php
# 此类是等会将实现反序列化操作的类
 class  test{
@unlink('p.phar');  //先删除已存在的phar, 因为phar重新生成不会清除掉之前以后的会追加。
$phar = new Phar('p.phar');   //创建一个 phar
$o = new test();       
$o -> data = 'Hello Li9hu';
$phar ->setStub('li9hu'.'__HALT_COMPILER();');    # 这行不写会自动生成文件表示, 下面会解释
$phar -> setmetadata($o);  //将我们的对象写入meta里边, 若不写入反序列化将没任何作用
$phar ->addFromString('1.test','hello')    # 要打包的字符串或者文件夹($phar -> buildFromDirectory(dirname(__FILE__)); ) 推荐使用字符串, 因为文件不存在会报错
                                           # 使用字符串的话参数1（1.test ）就是文件入口, 使用文件需要设置存根 setStub('文件名

   ?>

__HALT_COMPILER(); 为phar文件的标识, 没有将不会被phar识别会报错, 不写这一行测试会自动加上,手动设置的话可以伪装文件类型, 比如将如上的li9hu改为GIF89a 伪装成gif

命令行执行 php a.php 将会产生一个p.phar的文件

如下为拖进hex里查看到的数据,可以看到我们构造对象已使用序列化的方法写进去了。下面就只需要执行一个反序列化操作就能到到我们的目的了
